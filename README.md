# S:S — Steam / Nintendo Store リスト検索

Steam のウィッシュリスト・所持ゲームとNintendo Storeのお気に入りリスト・バーチャルゲームカードを相互検索し、価格・サムネイルを比較できるタイル一覧を作成する Chrome / Microsoft Edge 拡張機能です。WindowsとmacOS上のChromium系ブラウザーで同じ拡張機能ディレクトリを利用できます。

## インストール

1. Chromeでは `chrome://extensions`、Edgeでは `edge://extensions` を開き、「デベロッパー モード」を有効にします。
2. 「パッケージ化されていない拡張機能を読み込む」を選び、このリポジトリ内の `extension` フォルダを指定します。
3. Steam の[ウィッシュリスト](https://store.steampowered.com/wishlist/)・所持ゲーム、Nintendo Storeのお気に入りリスト、またはニンテンドーアカウントのバーチャルゲームカードを開きます。
4. ツールバーの拡張機能アイコンから「検索して一覧を作成」を押します。

実行前にポップアップの「取得件数」で 10件・50件・100件・全件、または任意の件数を選べます。

検索開始時には進捗ページが開きます。ツールバーのポップアップはChromeの仕様でフォーカスを失うと閉じるため、進捗確認と中断はこのページで行えます。

## 表示言語

- 拡張機能の詳細画面に表示される名前・説明は、Chrome / Edgeの表示言語に合わせて日本語または英語へ自動的に切り替わります。
- 手動で切り替える場合は、拡張機能詳細画面の「拡張機能のオプション」、または検索対象外ページでS:Sを起動した際の「表示言語」を使用します。
- 選択した言語は、起動ポップアップ、進捗画面、検索結果、通知へ反映されます。

## 対応環境

- Google Chrome（Windows / macOS）
- Microsoft Edge（Windows / macOS）

Manifest V3とChromium共通の `chrome.*` APIのみを使用しており、OS固有のパスやコマンドには依存しません。

## 動作

- Steam のアプリ ID、表示タイトル、価格（「カートに入れる」左隣の販売価格）、サムネイルを重複なく収集します。
- 各タイトルを `https://store-jp.nintendo.com/search/?q=...` で検索します。CORS制限を回避するため、検索ページをバックグラウンドタブで読み込み、ページ内から情報を取得します。商品リンクの現行形式（`/item/software/...`）と旧形式（`/products/...`）の両方に対応しています。
- Nintendo Store の検索候補をタイトルと価格の近さで順位付けし、検索カードの価格とサムネイルをそのまま利用します。検索失敗もエラー表示付きのカードとして一覧に残します。選択モードでは複数の商品を重複なく間隔を空けてカートに追加し、完了後にカートページを開きます。
- 一覧カードの左右のサムネイルから各ストアの商品ページを直接開けます。操作欄の `📋` は商品ページ、`🛒` はカート追加、`⭐` はSteamウィッシュリストまたはNintendo Storeのお気に入りへの追加です。追加処理はログイン状態を共有する専用ウィンドウで実行します。
- ヘッダーからカートまたはお気に入りのまとめ追加を選択できます。2つの選択モードと追加処理は排他的で、処理結果は単品ならゲーム名、まとめ追加なら成功件数を含む通知で知らせます。失敗または一部失敗時も警告通知とカード上のエラーを残します。
- 一覧カードは画面中央にタイル配置します。まとめ追加の選択モードでは、ラジオボタンだけでなくカードやサムネイルのクリックでも選択を切り替え、商品ページへの遷移は一時的に抑止します。
- Steamウィッシュリストへの追加は、商品ページ本体のログインセッションから旧UIの標準追加関数または新UIの追加ボタンを優先して実行し、公式追加APIを予備経路として使用します。
- Steamウィッシュリスト追加後は認証済みユーザーデータを再取得し、対象アプリIDが実際に反映された場合だけ成功通知を表示します。商品ページから相手ストアを単品検索したポップアップでも、Steamの「ウィッシュリストに追加」またはNintendo Storeの「ほしいものリストに追加」を実行できます。
- Steamのウィッシュリストで価格またはサムネイルを取得できない場合のみ、Steam商品ページを開いて情報を補完します。
- どちらの方向でも、検索カードのサムネイルまたは価格の片方が欠けていれば対象の商品ページから不足分だけを補完します。遅延読み込み画像、割引価格、無料表記にも対応します。
- 新しい拡張機能ページに、サムネイルとタイトルを持つカードをタイル表示します。見つからなかったタイトルやエラーになったタイトルは、ページ最下部に検索ページへのリンクと理由をまとめて表示します。

## 検索の優先順位

1. 商品IDの直接対応
2. 元タイトルの完全一致（Unicode正規化に加え、「嘘／噓」など意味が同じ異体字も統一）
3. 元タイトルの前方一致
4. 読み・エディション等を除外した正規化タイトル
5. 主題＋副題一致
6. ストア固有の別名・音写検索
7. タイトル中に明示された読み
8. Nintendo Store→Steamのみ、日本語題名に対する英語候補の安全な先頭候補

Steamでは、バンドル、サウンドトラック、DLC、デモ、テストプレイ等の正式版ではない候補を除外します。

Steam検索結果は複数行が段階的に表示されるため、最初の1件だけでは確定せず、候補集合が安定するか元タイトルの完全一致が表示されるまで待ってから照合します。「シナリオ配信」等の追加シナリオはDLC候補として除外します。

Nintendo Store→Steam検索では、キー不要のSteam Store JSONエンドポイントを先に利用して候補・種別・画像・価格を取得します。応答が利用できない場合だけ従来の専用検索ウィンドウと商品ページDOMへ自動的に戻ります。タイトル正規化とSteam除外規則は `extension/search-rules.js` の共通オブジェクトを、バックグラウンド・検索ページ・商品ページで共有します。

Steam検索結果が表示言語に合わせて翻訳され、商品URLのタイトル部分も `_` になっている場合は、検索1位の商品ページにある本体購入名を照合用の別表記として読み取ります。この別表記は新しい低優先度フォールバックではなく、第2順位の完全一致、第3順位の前方一致にだけ使用し、DLC・サウンドトラック・デモ等の除外判定を通過した本体に限って採用します。

カタカナだけで構成されたタイトルをNintendo StoreからSteamへ検索する場合は、元タイトルで0件だった後、第6順位の音写検索として共通の英単語・カナ読み辞書を最長一致で分割します。例えば `コンテント` + `ワーニング` を `content warning` に変換して再検索し、Steam側の完全一致・前方一致と除外判定を通過したゲームだけを採用します。これは商品IDやゲームタイトル全体の直接対応ではなく、両方向で再利用する単語単位の表記変換です。

Nintendo Store / Steam の画面構造が変わった場合は、`extension` 内の対応するコンテンツスクリプト（`wishlist.js`、`nintendo.js`、`steam-search.js`）のセレクターを更新してください。
